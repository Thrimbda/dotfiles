What Changed / What Was Decided

`axiom-cli.target` is the explicit non-graphical mode. It requires `multi-user.target`, wants `getty@tty1.service`, conflicts with `graphical.target`, and allows isolation.

`axiom-mode` wraps only fixed `systemctl` operations: set default target, isolate target, and show status. The command dispatch uses fixed case branches rather than user-controlled target names.

Risks and Limits

  • `systemctl isolate` terminates the graphical session immediately, so desktop work should be saved first.
  • Runtime isolate and power draw checks were not run in this environment because it is not live `axiom`.
  • Actual power savings depend on RTX 5090 idle behavior and display state after deployment.
