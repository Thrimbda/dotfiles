Reviewer Summary

本任务把 Axiom 本机 Rust CLI 从 axiom-mode 重命名为 axiomctl，并把正式入口调整为 axiomctl mode ...。原有的 cli、desktop、status 顶层工作流作为别名保留。

新增的 axiomctl reload 不是重写 hook 系统，而是固定 argv 调用现有 hey reload。这让 Axiom 有一个更合适的 host-control CLI 名字，同时避免把 broad dotfiles 工作流迁入 Rust。

Scope

In scope

  • Rename packages/axiom-mode to packages/axiomctl.
  • Update Axiom host package wiring and README usage.
  • Keep mode switching under fixed systemd targets.
  • Add a bounded reload bridge to existing hooks.

Out of scope

  • No broad hey rewrite.
  • No Rofi command migration.
  • No Caelestia desktop-control replacement.
  • No remote access or target policy changes.

What Changed / What Was Decided

axiomctl 现在是 Axiom host-control CLI。它拥有小而固定的行为集合：mode switching、mode status、reload bridge。hey 继续拥有 Nix/dotfiles 操作、动态脚本派发和 broader maintenance workflow。

Host 配置通过 pkgs.callPackage ../../packages/axiomctl 安装新包，并把 ${hey.binDir}/hey 注入给 reload bridge。Package 仍用 Nix 注入 systemctl store path。
