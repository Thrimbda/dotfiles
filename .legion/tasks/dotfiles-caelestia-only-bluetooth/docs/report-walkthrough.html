2 · Headless AuthAgent

启动边界

普通用户运行；环境、schema、session bus、GTK 与 icon 全部就绪后才建立 system-bus 行为。state=default 后才发送 READY。

单连接状态机

local export、RegisterAgent、RequestDefaultAgent 与 UnregisterAgent 使用同一 persistent Gio connection/sender。

不提供管理面

runner output 只有一个 public command，不加载 applet、PowerManager、KillSwitch、Menu、tray、manager 或 mechanism。

Fail closed

部分注册失败、未知 owner query、connection close、Release、BlueZ vanish/reappear 与信号退出都有显式逆序或本地清理。

3 · Ordinary / TLP rfkill 分流

共用 finalizer 只扫描 Bluetooth：无设备或已 unblocked 时零写；soft=1 时仅运行一次 rfkill unblock bluetooth 并重读验证。不执行 unblock all，不修改 hard block、WLAN persisted state 或 TLP state。

分支主机关键接线stock rfkill
OrdinaryAxiom / Azar / Harusame / Udon每个 systemd-rfkill invocation 有唯一 ExecStopPost finalizer；ADD 仍走 stock service；resume restart base helper保留 vendor service/socket
TLPRamenTLP weak-Wants base，base After TLP 且无 multi-user WantedBy；ADD 使用独立 @%k template；resume 走 post-resume继续 masked，无 ExecStopPost/InvocationID
 
multi-user.target → tlp.service → bluetooth-rfkill-unblock.service
                         weak Wants + helper After TLP

ADD    → bluetooth-rfkill-unblock@rfkillN.service
resume → post-resume.service → restart base helper

%k/%I 只决定 unit identity/Description，不进入 root ExecStart、Environment、shell 或 finalizer 参数。移除 base helper 的 multi-user WantedBy 后，旧 ordering cycle 的回边消失。

4 · Caelestia 命名策略

  1. 主列表最多五项：connected 优先，paired/bonded 次之，再优先真实名称。
  2. label 顺序为非 address 的 trimmed Alias → BlueZ deviceName → address。
  3. 大小写无关 label、规范化 address 与原始 index 提供确定性 tie-break；输入数组不被修改。
  4. 匿名且未连接/配对/bonded 的广播不占主列表；匿名资产与完整 pairing page 保留。
  5. 不根据 MAC、OUI 或广播猜测产品名。

生产 QML 与 Node fixture 使用同一个 policy source；installed policy 与仓库 source 逐字节一致。

9 · 组件级回滚

  1. Caelestia：只回退 package override、patch 与 policy。
  2. rfkill：回退对应 finalizer wiring；TLP 移除 weak-Wants/template/udev/post-resume。不要恢复 multi-user TLP edge、tlp-sleep hook 或旧 block→unblock workaround。
  3. AuthAgent：停新 unit，临时使用 bluetoothctl agent KeyboardDisplay；不要先恢复 stock Blueman。
  4. 整代：仅最后手段；它会恢复旧 surface/mechanism/rfkill 竞态。

不迁移/删除 BlueZ pairing 数据，不手工删除 store path。当前未部署，以上是 rollout 预案。

10 · Deploy-only hardware gates

Axiom / c1

  • 真实 UWSM/Wayland、GTK/schema/session/default/READY。
  • 真实 pairing local-only；Notify/history/state/journal 无敏感增量。
  • Release 与 BlueZ restart 恢复。
  • MediaTek late unbind/rebind、真实 rfkill/persisted state。
  • reboot 1 次、resume 2 次；WLAN 不变，Caelestia off→on。

Ramen / hlissner

  • reboot 无 cycle，TLP init 后 base InvocationID。
  • 至少两个 distinct tagged ADD template，base overlap 不合并。
  • resume 2 次，TLP resume 后 post-resume 新 base ID。
  • stock rfkill 始终 masked；WLAN 与真实 TLP state 不变。

其余主机

逐台确认 agent active/default、无旧 surface、ordinary finalizer effective 与 Bluetooth soft=0；未自然触发的 Agent1 类型继续作为 residual。
