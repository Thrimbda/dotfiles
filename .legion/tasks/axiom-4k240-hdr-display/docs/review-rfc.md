# RFC 审查：Axiom 4K 240Hz HDR 屏幕设置

## 结论

PASS。

## 审查结果

- 范围足够小：主线只要求把 Axiom 从当前 `3840x2160@60` 调整为 `3840x2160@240`，共享模块扩展仅用于表达可选 monitor 字段。
- HDR 优先级已明确低于 240Hz，且 RFC 没有要求撤销现有 `cm_enabled = false` 稳定性保护，因此不会把已知 DPMS/恢复崩溃风险直接推入默认配置。
- 回滚路径可执行：240Hz 不稳定时改回 `3840x2160@60` 或 `preferred`；HDR 后续实验不稳定时恢复关闭色彩管理并移除 HDR 字段。
- 验证路径可执行：Nix 求值可以覆盖配置生成形态，真实刷新率和 HDR 运行态则明确留给 Axiom 实机会话确认。

## 非阻塞建议

- 实现时不要为了 HDR 引入新的显示抽象；保持可选字段追加逻辑即可。
- 配置注释应避免宣称 Axiom 当前已启用 HDR，只说明 HDR 依赖重新开启 Hyprland 色彩管理并完成实机验证。
