# Aliyun Acorn HTTPS Firewall Ports

Status: ready for PR

## Summary

Supersedes the local-only staging shape from PR #112 by restoring public HTTPS for `vault.0xc1.space` and `status-axiom.0xc1.wang`, while keeping ACME disabled and public HTTP closed. It also adds firewall TCP ports `2223` and `2224`.

## Current Shape

- Firewall TCP ports: `[22,443,2222,2223,2224,2225,7000,34197]`.
- Public TCP `80` remains closed.
- `vault.0xc1.space` and `status-axiom.0xc1.wang` are nginx `onlySSL` vhosts on public `443`.
- ACME units are absent during staging.
- Missing staging certs are generated by `nginx.preStart` under `/var/lib/nginx-selfsigned/<domain>/` before nginx config validation.
- `nginx.service` owns `StateDirectory = "nginx-selfsigned"`, making that cert path writable under the service sandbox.
- Low-resource slimming from PR #112 remains intact.

## Reusable Decisions

- For `aliyun-acorn` staged auth-bearing vhosts, public `443` is required; public `80` is not.
- If ACME is intentionally disabled, provide an explicit non-store certificate/key path and generate temporary cert material on-host before nginx validates config.
- Self-signed staging certs are an operational bridge, not production trust material.

## Verification

- Toplevel build passed.
- Eval confirmed firewall ports include `443`, `2223`, and `2224`, while `80` is absent.
- Eval confirmed no generated ACME or Docker units.
- Generated nginx config shows public `443 ssl` listeners for both staged vhosts and no public `80` listener.
- Generated nginx preStart creates missing self-signed certs before `nginx -t`.

## Operational Follow-up

- After DNS points correctly, replace self-signed staging certs with ACME-managed certificates.
- After deployment, confirm remote switch succeeds, SSH remains responsive, nginx listens on public `443`, public `80` is closed, and cert files exist under `/var/lib/nginx-selfsigned` until ACME replaces them.
