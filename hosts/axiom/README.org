#+TITLE: Axiom
#+DATE:  May 14, 2026

* Overview
Axiom is C1's Ryzen 9950X + RTX 5090 NixOS workstation.

* Host Control CLI
Axiom provides a host-local ~axiomctl~ command for narrow machine-control tasks.
It switches between the full Hyprland desktop and an SSH-friendly CLI mode, shows
the relevant systemd target status, and can trigger the reviewed reload hook.

#+begin_src sh
axiomctl status
axiomctl mode cli
axiomctl mode desktop
axiomctl reload
#+end_src

~axiomctl mode cli~ sets the default systemd target to ~axiom-cli.target~ and
isolates it immediately. That target requires ~multi-user.target~, wants
~getty@tty1.service~, and conflicts with ~graphical.target~. SSH, reverse SSH,
Cloudflare Tunnel, and opencode remain available because they are normal
~multi-user.target~ services.

~axiomctl mode desktop~ sets the default target back to ~graphical.target~ and
isolates it immediately, allowing greetd/UWSM to start the Hyprland session.

~axiomctl reload~ delegates to the existing ~hey reload~ hook path. Broader
dotfiles and Nix workflows such as ~sync~, ~pull~, garbage collection, profile
management, and dynamic script dispatch remain owned by ~hey~.

* Keep Awake
Axiom does not enable Caelestia's built-in Keep Awake mode by default when the
Hyprland / Caelestia desktop session starts. Keep Awake remains available as
Caelestia's manual ~idleInhibitor~ toggle for temporary no-idle behavior.

Hypridle owns the repository idle timing and display power policy: it asks
Caelestia's ~WlSessionLock~ to lock after 15 minutes and turns DPMS off after
30 minutes. Caelestia's own shell idle monitors are aligned to the same 15 / 30
minute policy so the upstream 3 minute lock, 5 minute DPMS, and 10 minute sleep
defaults do not win first. Axiom no longer starts a separate session-scoped
sleep inhibitor or automatic idle sleep action by default.

The shell entrypoint is:

#+begin_src sh
caelestia shell idleInhibitor isEnabled
caelestia shell idleInhibitor enable
caelestia shell idleInhibitor disable
caelestia shell idleInhibitor toggle
caelestia shell lock lock
caelestia shell lock isLocked
#+end_src

The shell process itself is started by the Hyprland startup hook through the
generated ~caelestia-session~ runner, not as a ~systemd --user~ service. This
keeps Caelestia inside the graphical login session so local polkit rules apply
to shell-owned controls such as NetworkManager Wi-Fi actions.

The visible manual toggle is Caelestia's ~Keep Awake~ control. No separate
Axiom power mode launcher is used.

The session pre-start hook also narrows migration of the mutable shell config at
=~/.config/caelestia/shell.json= to Axiom-owned fields: launcher favorites and
~general.idle~. Other user shell settings are preserved.

Live checks:

#+begin_src sh
caelestia shell idleInhibitor isEnabled
#+end_src

This depends on the graphical Hyprland/Caelestia session. If Axiom is running
headless or no Hyprland session starts, this is not a pre-login system-wide
sleep policy. Direct sleep requests are not blocked by a repository-owned
sleep-inhibitor service.
