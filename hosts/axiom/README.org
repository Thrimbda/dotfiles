#+TITLE: Axiom
#+DATE:  May 14, 2026

* Overview
Axiom is C1's Ryzen 9950X + RTX 5090 NixOS workstation.

* Keep Awake
Axiom defaults to Caelestia's built-in Keep Awake mode when the Hyprland /
Caelestia desktop session starts. This reuses Caelestia's own ~idleInhibitor~
service, so the shell UI stays aligned with the requested keep-awake state.

Hypridle owns the idle lock and display power policy: it locks after 15 minutes
and turns DPMS off after 30 minutes. Axiom no longer starts a separate
session-scoped sleep inhibitor by default.

The shell entrypoint is:

#+begin_src sh
caelestia shell idleInhibitor isEnabled
caelestia shell idleInhibitor enable
caelestia shell idleInhibitor disable
caelestia shell idleInhibitor toggle
#+end_src

The shell process itself is started by the Hyprland startup hook through the
generated ~caelestia-session~ runner, not as a ~systemd --user~ service. This
keeps Caelestia inside the graphical login session so local polkit rules apply
to shell-owned controls such as NetworkManager Wi-Fi actions.

The visible toggle is Caelestia's ~Keep Awake~ control. No separate Axiom power
mode launcher is used.

Live checks:

#+begin_src sh
caelestia shell idleInhibitor isEnabled
#+end_src

This depends on the graphical Hyprland/Caelestia session. If Axiom is running
headless or no Hyprland session starts, this is not a pre-login system-wide
sleep policy. Direct sleep requests are not blocked by a repository-owned
sleep-inhibitor service.
